## Goal
Remove the in-memory (`memory`) persistence adapter and the runtime `--store`
toggle so the template ships PostgreSQL as its only persistence layer â€”
realizing the long-planned "PostgreSQL-only" direction (TECH_NOTES; sessions
006/007 open threads). The `todo.Repository` port must stay so integrators can
still add their own adapters.

## Outcome
Met. Shipped as **PR #9, squash-merged to `master` `8a46286`**. The in-memory
tier is gone; `--store` is removed and `--database-url` is now required (`serve`
fails fast without it). The port is unchanged. A second thread surfaced and was
resolved in the same PR: the repo violated the `go-testing` skill (no mockery,
hand-rolled fakes), so the session also adopted **mockery** for repository test
doubles. `moon run root:check` green (incl. the new `mockery-check`); OpenAPI
spec byte-identical; container-backed integration suite green against real
`postgres:17-alpine`. Implementation worktree removed, local `master`
fast-forwarded.

## Key Decisions
- **Port stays; drop one adapter (user explicit).** PostgreSQL is the only
  shipped adapter, but `todo.Repository` remains the extension seam â€” integrators
  implement it (or inject via `app.WithRepository`) without touching
  domain/transport. The abstraction did NOT collapse.
- **Adopt mockery + a small stateful fake** (user chose this over "mockery
  everywhere" and "hand-rolled fakes only"). The skill mandates mockery; the repo
  shipped none. The generated mock backs interaction/error tests; the fake backs
  end-to-end HTTP round-trips, where a scripted mock fits poorly (runtime-gen ID
  flows POSTâ†’GETâ†’Listâ†’complete).
- **`app.WithRepository` seam.** `TestAppWiring` was a no-DB unit test relying on
  the memory default; postgres-only makes `app.New` need a database. Rather than
  convert it to an integration test (or trim it), added a functional option that
  injects a repo and skips the connect â€” keeps the test fast AND doubles as the
  integrator extension point. (User picked this over the two alternatives.)
- **`TestServiceComplete` restructured to inject the existing todo** (FindByID
  returns it) instead of create-then-read, so every service-unit test is cleanly
  interaction/error-based and fits the mock â€” no stateful awkwardness left.
- **No-op stub repo for OpenAPI spec gen.** `OpenAPIYAML` previously leaned on
  `memory` to construct the service server-lessly; replaced with a tiny prod
  `noopRepository{}` (never persists), since the repo is never invoked during
  spec emit.
- **mockery drift-guard mirrors `sqlc-check`** (sed-swap `dir` into a temp config,
  `git diff --no-index`). The `// Code generated by mockery` header makes golangci
  auto-exclude the file; the pre-existing `testify/mock.Mock` golangci exclusion
  already covers the embedded type.

## Lessons
- **mockery v3 testify output is deterministic and version-stable.** The generated
  header carries no version/path/timestamp, so `mockery-check` is stable across
  patch bumps (same property as `sqlc-check`). Proto plugin: release assets are
  `mockery_{version}_{Darwin|Linux}_{x86_64|arm64}.tar.gz` â€” note `x86_64` (not
  `amd64`); map `aarch64â†’arm64`, keep `x86_64`.
- **A white-box test package can't import a package that imports the package under
  test.** Pointing `service_test.go` at `internal/todo/mocks` (which imports
  `internal/todo`) required switching it to the external `package todo_test` to
  avoid an import cycle; it only used exported identifiers, so the switch was clean.
- **Match the test double to what the test asserts.** A scripted mock is the wrong
  tool for a stateful create-then-read round-trip; a fake is the wrong tool for
  error injection. Restructuring `TestServiceComplete` to inject (not create) the
  existing todo was what let the mock cover it cleanly and kept the fake scoped to
  genuine end-to-end flows.
